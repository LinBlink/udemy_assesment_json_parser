<p><strong>What is </strong><code><strong>@PropertySource</strong></code><strong>?</strong><br>The <code>@PropertySource</code> annotation in Spring is used to declare a properties file that should be loaded into the Spring <code>Environment</code>. It allows externalized configuration by loading key-value pairs from a specified file into the application context.</p><p><strong>Key Features of </strong><code><strong>@PropertySource</strong></code><strong>:</strong></p><ol><li><p><strong>Loading External Properties:</strong></p><ul><li><p><code>@PropertySource</code> specifies the location of a properties file to be loaded into the Spring <code>Environment</code>.</p></li></ul></li><li><p><strong>Integration with </strong><code><strong>@Value</strong></code><strong>:</strong></p><ul><li><p>Once the properties file is loaded, individual properties can be accessed using the <code>@Value</code> annotation.</p></li><li><p>Example:</p><pre class=\"prettyprint linenums\">@Value(\"${my.property}\")\nprivate String myProperty;\n</pre></li></ul></li><li><p><strong>Used in Java Configuration:</strong></p><ul><li><p><code>@PropertySource</code> is typically used in Java-based configuration classes annotated with <code>@Configuration</code>.</p></li></ul></li><li><p><strong>Supports Multiple Files:</strong></p><ul><li><p>Multiple <code>@PropertySource</code> annotations can be used to load multiple properties files.</p></li></ul></li></ol><p><strong>Loading Properties with </strong><code><strong>@PropertySource</strong></code><strong>:</strong></p><pre class=\"prettyprint linenums\">@Configuration\n@PropertySource(\"classpath:custom.properties\")\npublic class AppConfig {\n\n    @Value(\"${custom.property}\")\n    private String customProperty;\n\n    @Bean\n    public MyBean myBean() {\n        System.out.println(\"Custom Property: \" + customProperty);\n        return new MyBean();\n    }\n}</pre><ul><li><p><strong>Key Points in the Example:</strong></p><ul><li><p>The <code>@PropertySource</code> annotation loads the <code>custom.properties</code> file from the classpath.</p></li><li><p>The <code>@Value</code> annotation is used to inject the value of <code>custom.property</code> into the <code>customProperty</code> field.</p></li></ul></li></ul><p><strong>Contents of </strong><code><strong>custom.properties</strong></code><strong>:</strong></p><pre class=\"prettyprint linenums\">custom.property=Hello, World!</pre><p><strong>Best Practices for Using </strong><code><strong>@PropertySource</strong></code><strong>:</strong></p><ol><li><p><strong>Use for Non-Standard Properties Files:</strong></p><ul><li><p>Use <code>@PropertySource</code> to load properties files that are not automatically handled by Spring Boot (e.g., files other than <code>application.properties</code> or <code>application.yml</code>).</p></li></ul></li><li><p><strong>Avoid Overusing </strong><code><strong>@PropertySource</strong></code><strong>:</strong></p><ul><li><p>For Spring Boot applications, prefer using <code>application.properties</code> or <code>application.yml</code> for configuration. Use <code>@PropertySource</code> only when necessary.</p></li></ul></li><li><p><strong>Use </strong><code><strong>@Value</strong></code><strong> or </strong><code><strong>Environment</strong></code><strong> to Access Properties:</strong></p><ul><li><p>Use the <code>@Value</code> annotation or the <code>Environment</code> object to access individual properties loaded via <code>@PropertySource</code>.</p></li></ul></li><li><p><strong>Handle Missing Files Gracefully:</strong></p><ul><li><p>Use the <code>ignoreResourceNotFound</code> attribute to avoid errors if the specified properties file is missing.</p></li><li><p>Example:</p><pre class=\"prettyprint linenums\">@PropertySource(value = \"classpath:optional.properties\", ignoreResourceNotFound = true)</pre></li></ul></li></ol>
